## Required DNS Records

### 1. A Record (Root Domain)
Points your main domain to Render's IP address

```
Type: A
Host: @ (or leave blank, or use "elitewealthcapita.uk")
Value: 216.24.57.1
TTL: Auto (or 3600)
```

**Purpose:** Makes elitewealthcapita.uk point to Render

---

### 2. CNAME Record (WWW Subdomain)
Points www subdomain to your Render service

```
Type: CNAME
Host: www
Value: my-site-ghnp.onrender.com
TTL: Auto (or 3600)
```

**Purpose:** Makes www.elitewealthcapita.uk point to Render

---

### Optional: CNAME for All Subdomains (Wildcard)
If you want all subdomains to work (e.g., api.elitewealthcapita.uk)

```
Type: CNAME
Host: *
Value: my-site-ghnp.onrender.com
TTL: Auto (or 3600)
```

**Purpose:** Makes *.elitewealthcapita.uk point to Render

## If Creating NEW Render Service

**Important:** Your new service will have a DIFFERENT URL!

Example old: `my-site-ghnp.onrender.com`
Example new: `elite-wealth-xyz.onrender.com`

**You MUST update your CNAME record** to point to new service URL:

```
Type: CNAME
Host: www
Value: elite-wealth-xyz.onrender.com (NEW URL)
```

**How to find your new service URL:**
1. Go to Render dashboard
2. Click your new service
3. Look for "Service URL" at top
4. Copy the `.onrender.com` URL
5. Use this in your CNAME record

## Troubleshooting DNS Issues

### Site shows "Page not found" or "Service unavailable"
- **Cause:** DNS not propagated yet
- **Fix:** Wait 30 more minutes, clear browser cache

### Site shows "Certificate error" or "Not secure"
- **Cause:** SSL certificate still provisioning
- **Fix:** Wait 10-15 minutes after adding domain

### www.domain.com works but domain.com doesn't (or vice versa)
- **Cause:** Missing A or CNAME record
- **Fix:** Add both A record (@) and CNAME (www)

### "This site can't be reached" error
- **Cause:** DNS pointing to wrong IP
- **Fix:** Verify A record = 216.24.57.1

### Mixed content errors
- **Cause:** Site loading HTTP resources on HTTPS page
- **Fix:** All resources should use HTTPS in templates
